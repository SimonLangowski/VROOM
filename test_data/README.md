# test_data

Reference integers and RNS residues for correctness tests. Values are generated by `scripts/rns_secp256k1.py` (or checked against GMP) and included as headers so tests run without a Python runtime.

## Files

| File | Curve / use | Consumed by |
|------|-------------|-------------|
| `test_values_bls12_381.hpp` | BLS12-381 G1 field (`a`, `b`, `a*b mod p`, m1/m2 residues) | `test_bls12_381_field_mul`, `examples/02_singular_modmul` |
| `test_values.hpp` | BLS12-381 (52-bit Matrix ring layout) | `gmp_wrapper.hpp`, general field tests |
| `test_values_50.hpp` | BLS12-381 50-bit parameters | 50-bit pairing / reduction tests |
| `test_values_bn254.hpp` | BN254 field + moduli / rotation constants | BN254 field and IntRNS4 tests |
| `test_values_bn254_ec.hpp` | BN254 EC point cases | `test_ec_bn254` |
| `test_points.hpp` | EC affine points | EC addition / doubling tests |
| `pairing_values.hpp` | Pairing inputs / outputs | `test_pairing`, `test_bls12_381_pairing` |
| `fp12ring_test_values.hpp` | FP12 tower elements | FP12 / final-exponentiation tests |

Each file typically provides decimal-string moduli (`modulus`, `a`, `b`, `c_correct`) and optionally precomputed RNS limb arrays (`a_m1`, `a_m2`, …) for cross-checking Python and C++ paths.

## Regenerating

From `scripts/`:

```bash
python rns_secp256k1.py
```

This refreshes `scripts/*_intrns4_params.hpp` and can rewrite the `test_values_*.hpp` files (see functions at the bottom of `rns_secp256k1.py`). Precomputed C++ ring wiring lives under `src/` — see **ARTIFACT.md** § Parameter generation.
